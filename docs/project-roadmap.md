# Project Roadmap

Last audited: 2026-05-31

This document tracks what still needs to be done across product, public content, admin workflow, research operations, and documentation. It is meant to sit above the generated research queue and below the product brief.

## Current State

The project is a working alpha of the Hallmarks of Aging LEV Tracker.

- The public app builds and has routes for overview, hallmarks, tracks, interventions, studies, findings, activity, methods, state-of-the-field notes, and admin review.
- The homepage implements the current wireframe direction: overall outlook, hallmark grid, recent changes, 2036 scenario lens, and trust notes.
- The file-backed admin review workflow exists for candidate bundles, review comments, evidence reviews, promotion readiness, and publication events.
- The research planning layer is active: `research/state/coverage-status.v1.json` and `research/backlog/track-priority.v1.json` are regenerated by `npm run sync:research-planning`.
- Track coverage is still early: 11 of 38 seeded tracks have baseline coverage, and 27 remain unstarted.

## Priority 3: Data and Schema Hardening

1. Add validation commands for JSON records.
   - Validate all files under `data/`, `examples/`, `taxonomies/`, and `research/` against schemas.
   - Make validation part of the normal verification checklist.
   - Current status: partially implemented for candidate-bundle workflows with `npm run research:bundle -- validate --bundle <bundle-id>`.
   - Remaining: broad schema validation across all JSON collections.

2. Add app-level schemas where the public app now has implicit content contracts.
   - Scenario lens
   - State-of-the-field editions
   - Homepage aggregate summaries
   - Hallmark insight records

3. Add source ingestion rules.
   - PubMed
   - ClinicalTrials.gov
   - Manual curator entry

4. Normalize intervention records.
   - The schema exists, but the public app currently relies mostly on IDs and track exemplars.
   - Intervention records are needed before intervention detail pages can be useful.
